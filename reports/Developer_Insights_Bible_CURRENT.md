## 1) Project Overview

**The Attention Economy** is a multi-page Streamlit dashboard that tracks
14 major media/tech companies (Alphabet, Amazon, Apple, Meta, Microsoft,
Netflix, Disney, Comcast, Spotify, Roku, Warner Bros. Discovery, Paramount,
Snap, Pinterest) across revenue, segments, subscribers, stock performance,
and macro advertising market data.

**Primary data source:** A single Excel workbook (`Earnings + stocks copy.xlsx`)
stored in `app/attached_assets/`. This file is ~17MB and tracked in git.
The workbook contains 40 sheets covering financials, segments, macro data,
country-level advertising, stock prices, and transcripts.

**Secondary data source:** Google Sheets (same workbook exported as XLSX).
Used as fallback only when no valid local file is detected.

**Intelligence layer:** SQLite database (`earningscall_intelligence.db`)
built from earnings call transcript .txt files via the intelligence pipeline.
Powers the transcript quotes strip (Human Voice) and Genie AI chat.

## 7) Data Flow: Excel → Processors → Pages

```
Excel Workbook (app/attached_assets/*.xlsx)
  │
  ├─ resolve_financial_data_xlsx()        ← utils/workbook_source.py
  │    Resolution order:
  │    1. Explicit local candidates passed by caller
  │    2. Auto-detect *.xlsx in app/attached_assets/
  │    3. Download from GOOGLE_SHEET_URL (20s timeout)
  │
  ├─ FinancialDataProcessor               ← app/data_processor.py
  │    .load_data()  →  reads sheets:
  │      Company_metrics_earnings_values  → df_metrics
  │      Company_yearly_segments_values   → df_segments
  │      Company_subscribers_values       → df_subscribers
  │      Company_Employees                → employee data
  │      Company_revenue_by_region        → regional data
  │    Cached via get_data_processor() in utils/state_management.py
  │
  ├─ _read_excel_sheet_cached()           ← Welcome.py internal
  │    Reads individual sheets on demand (KPIs, map, anatomy, etc.)
  │    Cached with @st.cache_data(ttl=4h)
  │
  ├─ load_quarterly_segments()            ← 01_Earnings.py internal
  │    Reads per-company 'X Quarterly Segments' sheets
  │
  └─ load_combined_stock_market_data()    ← utils/workbook_market_data.py
       Reads 'Stocks & Crypto' sheet
       → Used by Welcome.py (market tape) and 02_Stocks.py

SQLite (earningscall_intelligence.db)
  Built by: scripts/build_intelligence_db.py
  Tables: transcript_highlights, transcript_topics, topic_metrics
  Used by: Welcome.py (Human Voice strip), 04_Genie.py (Genie AI context)
```

## 9) Transcript Intelligence Pipeline

**Orchestrator:** `scripts/sync_all_intelligence.py`

**Input:** Earnings call transcripts as `.txt` files in `earningscall_transcripts/`
**Output:** SQLite DB + CSV files consumed by Home page and Genie AI

- `run_step("1/7 Rebuild transcript index", [py, "scripts/rebuild_transcript_index.py", "--root", args.root], repo_root)`
- `run_step("2/7 Extract topics + metrics", [py, "scripts/extract_transcript_topics.py", "--root", args.root], repo_root)`
- `run_step("3/7 Extract KPI values", [py, "scripts/extract_kpi_values.py", "--root", args.root], repo_root)`
- `run_step("4/7 Extract transcript highlights", [py, "scripts/extract_transcript_highlights_from_sheet.py", "--out-dir", args.root], repo_root)`
- `run_step("5/7 Build intelligence DB", [py, "scripts/build_intelligence_db.py", "--db", args.db, "--root", args.root], repo_root)`
- `run_step("6/7 Sync workbook metrics to SQL", [py, "scripts/sync_gsheet_to_sql.py", "--db", args.db], repo_root)`
- `run_step("7/7 Generate summary insights", [py, "scripts/generate_insights.py", "--db", args.db], repo_root)`

**Transcript CSVs in `earningscall_transcripts/`:**
- `transcript_index.csv` (present): Master index: company/year/quarter/file path
- `transcript_topics.csv` (present): Topics extracted per transcript
- `transcript_kpis.csv` (present): KPI values mentioned in transcripts
- `transcript_highlights.csv` (present): Key quote highlights per transcript
- `overview_iconic_quotes.csv` (present): Hand-picked / top-ranked quotes for Overview
- `topic_metrics.csv` (present): Aggregated topic frequency metrics
- `generated_insights_latest.csv` (present): AI-generated insight text (latest run)

**Trigger:** Pipeline only runs automatically if `AUTO_REFRESH_INTELLIGENCE_PIPELINE_ON_STARTUP=1`.
Otherwise run manually: `python3 scripts/sync_all_intelligence.py`

## 15) Known Operational Risks

- **CSS bleeding (global selectors):** Welcome.py injects CSS using `.stApp`, `body`, `.main` — these apply globally. Earnings.py charts now use `render_plotly(light_theme=False)` by default (dark theme). If Earnings text appears invisible, check `render_plotly()` in 01_Earnings.py.
- **M2 sheet dual naming:** Old workbooks have sheet `M2_values` with columns `USD observation_date`/`WM2NS`. New workbooks use sheet `M2` with columns `observation_date`/`M2SL`. All 4 load points now try both names via try-loop.
- **Overview complexity:** `00_Overview.py` is >10k lines with many branch `st.stop()` exits — hard to reason, easy to regress.
- **HF cold start latency:** Free HF tier: ~30–60s container wake (unavoidable). Data load itself is ~2s via local XLSX.
- **SQLite DB not in git:** `earningscall_intelligence.db` is gitignored. Genie AI and Human Voice strip fail silently if DB missing.
- **No auto-regeneration of bible:** Bible does NOT auto-regenerate. Must run `python3 scripts/refresh_dev_manual.py` manually after every change to app/, scripts/, or .streamlit/.
- **Unused utility drift:** `app/utils/` has ~40 modules; several may be legacy. `insights_loader_fixed.py`, `optimized_data_loader.py` look like duplicates.

## 18) Regeneration & Artifact Policy

**To regenerate this bible after any code change:**
```bash
python3 scripts/refresh_dev_manual.py
```
This runs:
1. `scripts/generate_dev_manual_assets.py` — PNG charts
2. `scripts/generate_current_bible.py` — this markdown file
3. `scripts/generate_current_bible_pdf.py` — PDF export
4. Removes legacy duplicate files
5. Writes `reports/DEV_MANUAL_STATUS.json`

**Canonical outputs (never recreate old filenames):**
- `reports/Developer_Insights_Bible_CURRENT.md`
- `reports/Developer_Insights_Bible_CURRENT_Full.pdf`

**Legacy files that must NOT be recreated:**
- `reports/Developer_Insights_Bible.md`
- `reports/Developer_Insights_Bible_Full.pdf`
